# 07. UI/UX 설계 (UI/UX Design)

> 원칙: **변화(diff)를 한눈에.** 사용자는 "로그를 검색"하러 오지 않는다. "이번 배포가 안전한가"를 알러 온다.
> 화면 수를 최소화하고(2~3개), 데모에서 90초 안에 가치가 전달되게 한다.

---

## 1. 화면 구성 (3개면 충분)

```
1. 배포 목록 (대시보드 홈)      /            → 배포들의 안전/위험 한눈에
2. 배포 상세 + Drift 뷰        /deployments/:id  → 무엇이 달라졌나 (핵심 화면)
3. (옵션) 배포 등록 모달        -            → 데모용 트리거
```

---

## 2. 화면 1 — 배포 목록 (홈)

각 배포를 **severity 색상 신호등**으로 표시. 위험한 배포가 즉시 눈에 띈다.

```
┌──────────────────────────────────────────────────────────────────┐
│  🔍 LogLens          "배포 후 무엇이 바뀌었나"        [+ 배포 등록]  │
├──────────────────────────────────────────────────────────────────┤
│  서비스: [payment-api ▼]   환경: [production ▼]                    │
├──────────────────────────────────────────────────────────────────┤
│  ● CRITICAL   payment-api   a1b2c3d   2분 전     drift 72  [보기 →] │
│     └ 신규 에러 패턴 3종 · 에러율 4%→18%                            │
│  ─────────────────────────────────────────────────────────────── │
│  ● SAFE       payment-api   9f8e7d6   18시간 전  drift  6  [보기 →] │
│  ● WARNING    auth-service  7c6d5e4   1일 전     drift 31  [보기 →] │
│  ● SAFE       payment-api   5a4b3c2   2일 전     drift  3  [보기 →] │
└──────────────────────────────────────────────────────────────────┘
```

**핵심 요소**
- 좌측 **상태 점(●)**: 🔴 critical / 🟡 warning / 🟢 safe
- drift 점수 뱃지(0~100)
- 한 줄 요약(가장 중요한 변화)
- 최신 배포가 맨 위, critical은 시각적으로 강조

---

## 3. 화면 2 — 배포 상세 + Drift 뷰 (핵심)

상단에 **판정(verdict)**, 중단에 **AI 요약**, 하단에 **변경 패턴 diff**.

```
┌──────────────────────────────────────────────────────────────────┐
│  ← 목록   payment-api  ·  a1b2c3d  ·  production  ·  2분 전        │
├──────────────────────────────────────────────────────────────────┤
│                                                                    │
│   🔴  CRITICAL          Drift Score  72 / 100                      │
│   ┌──────────────────────────────────────────────────────────┐   │
│   │  ⚠️  즉시 롤백을 권고합니다.                                │   │
│   │     결제 경로의 신규 에러와 에러율 4.5배 상승은            │   │
│   │     사용자 영향이 큽니다.       [ 롤백으로 표시 ]          │   │
│   └──────────────────────────────────────────────────────────┘   │
│                                                                    │
│   📝 AI 요약                                                       │
│   이번 배포 후 결제 모듈에서 새로운 NullPointerException 에러가     │
│   142건 발생했고, DB timeout 로그가 배포 전 대비 70배 급증했습니다. │
│   전체 에러율은 4% → 18%로 상승했습니다.                           │
│                                                                    │
│   📊 지표 변화                                                     │
│   에러율    4% ──────▶ 18%   🔴 +14%p                             │
│   총 패턴   120  ────▶ 126   (+6 신규)                            │
│                                                                    │
├──────────────────────────────────────────────────────────────────┤
│  무엇이 달라졌나 (Drift Detail)                                    │
│                                                                    │
│  🆕 신규 패턴 (3)                                                  │
│   🔴 NullPointerException at <PATH>            142건  [error]      │
│   🔴 payment gateway returned <NUM>             88건  [error]      │
│   🟡 retrying request <NUM>/3                   54건  [warn]       │
│                                                                    │
│  📈 급증한 패턴 (1)                                                │
│   🔴 DB timeout after <NUM>ms       3 ▶ 210건  (70배 ↑)           │
│                                                                    │
│  📉 사라진 패턴 (1)                                                │
│   ⚪ Cache warmed in <NUM>ms         88 ▶ 0건  (소멸)             │
│                                                                    │
└──────────────────────────────────────────────────────────────────┘
```

**UX 포인트**
- **판정을 맨 위에**: 스크롤 없이 "롤백해야 하나?"의 답이 보인다.
- diff는 GitHub PR 처럼 친숙한 +/- 메타포 사용(🆕 신규 / 📈 급증 / 📉 소멸).
- 에러 레벨 패턴은 빨강으로 강조 → 눈이 위험으로 먼저 간다.
- 숫자 변화는 `before ▶ after` 화살표로 항상 "변화"를 보여준다(절대값 아님).

## 4. 화면 3 — 배포 등록 모달 (데모 트리거)

```
┌─────────────────────────────────┐
│  새 배포 등록                    │
│  서비스   [ payment-api      ]   │
│  버전     [ a1b2c3d          ]   │
│  환경     [ production    ▼  ]   │
│           [ 취소 ]  [ 등록 ]     │
└─────────────────────────────────┘
```

데모에서는 등록 → (시드/시뮬레이터가 로그 ingest) → "분석 실행" 버튼 → 상세 화면 이동 흐름.

## 5. 디자인 시스템 (해커톤 최소)

| 요소 | 선택 |
|------|------|
| 프레임워크 | Next.js 15 App Router + React Server Components |
| 스타일 | **Tailwind CSS** |
| 컴포넌트 | **shadcn/ui** (Card, Badge, Button, Table) — 빠른 완성도 |
| 아이콘 | lucide-react |
| 차트 | 최소화 — 숫자 + 화살표로 충분. 필요시 recharts 막대 1개 |

### Severity 색상 토큰
```css
--safe:     #16a34a;  /* green-600  🟢 */
--warning:  #d97706;  /* amber-600  🟡 */
--critical: #dc2626;  /* red-600    🔴 */
```

## 6. 컴포넌트 분해 (구현 가이드)

```
app/page.tsx                         # 배포 목록 (서버 컴포넌트, DB 직접 조회)
  └ <DeploymentRow severity score /> # 신호등 행
app/deployments/[id]/page.tsx        # 상세 (서버 컴포넌트)
  ├ <VerdictBanner severity recommendation />
  ├ <AiSummary text />
  ├ <MetricDelta label before after />
  └ <DriftSection title items kind />   # 신규/급증/소멸 공용
components/
  ├ SeverityBadge.tsx
  └ NewDeploymentModal.tsx (client)
```

> 데이터 패칭은 서버 컴포넌트에서 DB를 직접 읽어(`lib/db`) 단순화. 클라이언트 컴포넌트는 모달·버튼 상호작용에만.

---

## 7. UX 원칙 요약

1. **결론 먼저**: severity와 롤백 권고를 화면 최상단에.
2. **변화만 보여준다**: 모든 수치는 `before ▶ after`.
3. **위험은 빨강**: 에러 패턴/에러율 상승은 색으로 즉시 구분.
4. **클릭 최소화**: 목록 → 상세 2클릭 안에 모든 인사이트.
5. **로그 검색 UI 없음**: 우리는 검색 도구가 아니다. drift 도구다.

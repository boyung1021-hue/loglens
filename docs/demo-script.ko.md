# LogLens — 데모 영상 스크립트 (한국어)

> 목표 길이: 약 2분. 내레이션 + 화면 동작.
> 아래 대사는 가이드입니다 — 자연스럽게 본인 말투로 풀어서 말하세요.
> (화면 UI는 영어이므로 용어는 영어 그대로 인용합니다.)

---

## 0. 녹화 전 준비

```bash
# 데모 모드로 실행 (DB 없이 깔끔한 영어 샘플 데이터)
DATABASE_URL= pnpm dev
# 접속: http://localhost:3000
```

- 브라우저 배율을 **125~150%** 로 키워 텍스트가 잘 보이게 하세요.
- 상단에 🧪 **Demo mode** 배너가 보이면 정상입니다.
- 샘플 데이터: 서비스 `payment-api`, **배포 4건** (정상 3 + 최신 CRITICAL 1).

## 1. 내레이션 스크립트 (~2분)

### ① 오프닝 — 문제 제기 (0:00–0:20)
**화면:** 홈 화면(배포 목록)

> "배포를 올린 직후엔 로그가 폭증합니다. 그런데 정작 어려운 건 —
> 이게 **이번 배포가 새로 만든 에러**인지, 아니면 원래 있던 노이즈인지
> 구분하는 일이죠. 로그를 한 줄씩 읽는 방식으론 감당이 안 됩니다.
> **LogLens**는 이걸 자동으로 답해줍니다. 배포 전후의 로그 **패턴**을
> 비교해서, 롤백이 필요한지 바로 알려줍니다."

### ② 목록 화면 — 한눈에 보는 상태 (0:20–0:40)
**화면:** 배포 목록의 배지들을 마우스로 훑기

> "한 서비스의 배포 타임라인입니다. 각 배포마다 **drift score**와
> **severity**가 신호등처럼 표시됩니다. 이전 배포 3건은 모두 **SAFE** 인데,
> 가장 최근 배포는 점수 **92**, **CRITICAL** 로 잡혔네요. 바로 열어보겠습니다."

### ③ 상세 화면 — 핵심 (0:40–1:30)
**화면:** CRITICAL 배포(`@a1b2c3d`) 클릭 → 상세 페이지

> "drift score **92점**(100점 만점). 지표를 보면 **error rate가 0%에서 4%로**
> 뛰었고, 구분되는 로그 패턴 종류도 5개에서 8개로 늘었습니다."

*(AI Summary 박스를 가리키며)*

> "**AI summary**가 이 변화를 자연어로 정리하고, 명확하게 결론을 줍니다 —
> **즉시 롤백을 검토하라**고요. 이 요약은 결정적(rule-based) 엔진 위에 얹힌
> 레이어라서, **AI 키 없이 오프라인에서도** 그대로 동작합니다."

*(New patterns 섹션으로 스크롤)*

> "무엇이 위험한지도 콕 짚어줍니다. 이번 배포에서 **새로 등장한 에러 패턴** —
> NullPointerException, DB timeout, payment gateway 오류 — 이게 점수를
> 끌어올린 주범입니다. 반대로 이전엔 있던 정상 패턴 하나는 **사라졌는데
> (Disappeared)**, 기능 중단 신호일 수 있습니다."

### ④ 대조 — 정상 배포는 어떻게 보이나 (1:30–1:50)
**화면:** "← Deployments" 클릭 → **SAFE** 배포 하나 열기

> "비교를 위해 정상 배포를 보면, drift score **0** —
> *no significant drift, this looks like a healthy deployment*.
> 변화가 없을 땐 조용합니다. 즉, **시끄러워야 할 때만 시끄럽게** 알려주는 거죠."

### ⑤ 마무리 — 동작 원리 + 한 줄 정리 (1:50–2:10)
**화면:** 다시 홈 화면

> "동작 원리는 간단합니다. LogLens는 로그를 **패턴으로 정규화**하고,
> 배포 **전과 후**를 비교해서 **신규·급증·급감·소멸**을 분류하고 점수를 냅니다.
> 원본 로그는 저장하지 않아 가볍고 안전합니다. **배포 직후 30초,
> 롤백 판단을 자동화하는 LogLens였습니다.**"

> "배포 직후엔 로그가 폭증하죠. 근데 이게 *새로 생긴* 에러인지 그냥 노이즈인지
> 구분이 어렵습니다. **LogLens**는 배포 전후의 로그 **패턴**을 비교해 롤백
> 여부를 알려줍니다.
>
> 한 서비스의 배포 타임라인입니다 — 정상 3건, 그런데 최신 배포는 점수
> **92, CRITICAL**. 안을 보면 **error rate가 0%에서 4%로** 뛰었고,
> **AI 요약이 즉시 롤백을 권고**합니다. 원인까지 짚어주죠 —
> NullPointerException, DB timeout 같은 신규 에러 패턴. 반대로 정상 배포는
> 점수 **0**, 조용합니다.
>
> LogLens는 로그를 패턴으로 정규화하고, baseline과 current를 비교해
> 롤백 판단을 자동화합니다 — 배포 직후 30초 안에요."

## 4. 토킹 포인트 / Q&A 대비

- **왜 원본 로그가 아니라 패턴인가요?** 가변값(ID·타임스탬프·소요시간)을 토큰으로
  정규화해 "같은 종류의 로그"를 하나의 패턴으로 묶습니다. 노이즈가 사라지고
  배포 전후 비교가 가능해집니다.
- **점수는 어떻게 계산되나요?** 결정적(rule-based) 엔진이 신규 에러 패턴과
  에러율 상승에 가장 큰 가중치를 둬서 0–100으로 산출합니다. AI는 그 위에
  얹는 자연어 요약 레이어일 뿐입니다.
- **AI/API가 죽으면요?** 규칙 기반 `fallbackSummary`로 떨어집니다 —
  데모(와 운영)는 절대 멈추지 않습니다.
- **프라이버시는요?** 원본 로그 라인은 저장하지 않고, 정규화된 패턴 통계만
  남깁니다.
- **severity 임계값:** `safe` < 25 ≤ `warning` < 60 ≤ `critical`.
